# Skill: Pickup Preflight

Use before executing a `go` work unit.

## Steps
1. Re-read `backlog/index.md`.
2. Find the first row in index order where your current role appears in `Next Eligible Role(s)`.
3. Re-read the selected issue file `backlog/items/PB-*.md`.
4. Confirm role gate and item status are consistent with index row.
5. If mismatch exists, create a blocker/status comment in the issue and fix index drift in the same work unit (or ask stakeholder if ambiguous).

## Output
- Active role
- Selected backlog item
- Reason this item is eligible now
