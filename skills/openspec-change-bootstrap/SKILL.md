# Skill: OpenSpec Change Bootstrap

Use when a backlog item requires non-trivial design/change packaging.

## Steps
1. Create change directory:
   - `openspec/changes/chg-<pb-id>-<short-title>/`
2. Copy templates:
   - `proposal.md`
   - `design.md`
   - `tasks.md`
   - `spec-delta.md`
3. Fill minimum required fields:
   - backlog id
   - problem statement
   - proposed change
   - role-based task list
4. Add backlink in the backlog issue comment log to the OpenSpec path.

## Output
- A complete OpenSpec change package linked from backlog issue.
