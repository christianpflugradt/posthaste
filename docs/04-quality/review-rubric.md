# Review Rubric

## Severity Levels
- P0: Release-blocking correctness/safety issue.
- P1: High-impact bug or regression risk.
- P2: Moderate issue affecting quality or maintainability.
- P3: Minor issue or improvement suggestion.

## Review Checklist
- Correctness of feature behavior.
- Regression risk and edge cases.
- Test coverage adequacy.
- Evaluation evidence for AI behavior changes.
- Security/safety concerns.
- Documentation completeness.

## Required Review Output
- Findings ordered by severity.
- File/line references where applicable.
- Clear disposition: block, required follow-up, or approved.
